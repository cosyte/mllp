Phase 8 residuals (MLLP-8.1): fixed the pre-existing crash where a bind error (`EADDRINUSE`, `EACCES`, …) on a server with **no** `'error'` listener crashed the process instead of rejecting the `listen()` promise — the internal `net.Server`/`tls.Server` error forwarder is now guarded by server state, with the contract documented: listener attached ⇒ always forwarded; no listener ⇒ a bind-window error rejects `listen()` (the primary surface), a stale post-`close()` error is dropped, and a runtime error **while serving** (accept-loop `EMFILE`) deliberately keeps Node's fail-loud crash so a silent accept outage is impossible. A throwing `'listening'`/`'securityWarning'` subscriber can no longer strand the `listen()` promise or wedge the single-flight guard (throw contained, surfaced via the guarded `'error'` tap, `listen()` resolves). Consolidated `listen()`'s five settle paths into one idempotent first-caller-wins helper — including dropping the abort listener before the `'listening'`/`'securityWarning'` emits, so an abort fired from inside those handlers is deliberately too late (the bind wins; use `close()`) and can never strand `listening: true` on a closed socket. Documented that the post-bind wildcard-reject window is accept-safe, clarified the two-tier `allowWildcardBind` enforcement JSDoc (literal spellings pre-bind, resolver-only shorthands post-bind), and qualified `close()`'s proactive-settle claim: an already-aborted `close({ signal })` is a no-op `AbortError` rejection that leaves an in-flight `listen()` to settle on its own bind outcome.
