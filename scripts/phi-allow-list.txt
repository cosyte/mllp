# scripts/phi-allow-list.txt
#
# PHI-scan allow-list, the positive declaration that a fixture's identifiers
# are synthetic. `scripts/phi-scan.ts` reads this file; any realistic-PHI-shaped
# token NOT covered here is a hit.
#
# mllp is a TRANSPORT/framing library: it wraps HL7 v2 messages in MLLP frames
# (`VT + payload + FS CR`). Its data fixtures are therefore MLLP-framed HL7 v2
# messages (`test/**/*.frame.bin`), whose PHI shapes are IDENTICAL to `@cosyte/hl7`'s.
# The scanner strips the MLLP frame, then runs the same HL7 v2 segment/field-aware
# detection hl7 uses. A framed binary fixture is byte-strict at the front (the VT
# start-block, then the MSH segment), so it cannot carry an inline `# synthetic:
# true` header, that would break every framing test. This allow-list is therefore
# the synthetic-fixture declaration mechanism, the same approach `@cosyte/hl7`,
# `@cosyte/dicom` (binary `.dcm`) and `@cosyte/x12` (byte-strict `.edi`) use.
#
# Format: `<TAG> <value>` per line. `#` comments and blank lines are ignored.
#   NAME <token>          synthetic person-name word (XPN / XCN name components +
#                         site-defined Z-segment names), matched case-insensitively
#   DOB <YYYYMMDD|YYYY>   synthetic date of birth (PID-7 / NK1-16), normalized
#   ADDR <street line>    synthetic street address (XAD component 1, e.g. PID-11),
#                         matched case-insensitively
#   ID <value>            synthetic id that legitimately matches an SSN / bare-MRN
#                         (6-9 digit) shape (PID-19, CX SS-typed, PID-3/-18)
#   EMAILDOMAIN <domain>  reserved / test email domain that is allowed to appear
#
# To add a new synthetic fixture: reuse the tokens below, or append new ones here
# and commit. Adding a token is a reviewed, intentional act, never silent.
