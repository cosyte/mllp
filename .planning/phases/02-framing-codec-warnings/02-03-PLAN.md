<pitfall_ref>
<!-- From .planning/research/PITFALLS.md Pitfall 1: Delimiter split across chunk boundary -->
The #1 naive bug: byte loop that looks for VT/FS/CR using string operations or by processing whole-buffer patterns rather than byte-by-byte FSM. A chunk may end mid-frame (e.g., [VT, M, S], then [H, |...], then [..., FS], then [CR]). The only correct approach is a byte-by-byte FSM with a dynamic accumulator buffer that grows as payload bytes arrive.

The FSM state transitions are:
- SCANNING_FOR_VT:
  - byte == VT → emit MLLP_MISSING_LEADING_VT warning (if state not initial and not after frame), transition to READING_PAYLOAD
    Wait — correct: in SCANNING_FOR_VT, a VT byte IS the desired byte; transition to READING_PAYLOAD.
    Non-VT bytes before VT depend on tolerance opts.
  - With allowLeadingWhitespace: SP(0x20), TAB(0x09), LF(0x0A), CR(0x0D) before VT → MLLP_LEADING_WHITESPACE warning
  - With allowMissingLeadingVt: first non-whitespace, non-VT byte starts payload accumulation → MLLP_MISSING_LEADING_VT warning
  - Any other non-VT byte without tolerance → MllpFramingError('MLLP_MISSING_LEADING_VT')
  - But: MLLP_TRAILING_BYTES and MLLP_EMPTY_PAYLOAD are always warnings, never errors.
    Trailing bytes between CR and next VT emit MLLP_TRAILING_BYTES (always a warning).

- READING_PAYLOAD:
  - Regular byte (not VT/FS) → append to accumulator
  - FS byte → check accumulator size == 0? emit MLLP_EMPTY_PAYLOAD warning; transition to EXPECTING_CR
  - VT byte mid-payload → this is unusual; treat as unexpected (MLLP_PAYLOAD_CONTAINS_VT isn't a decoder tolerance but the decoder should handle it — actually the encoder guards this; the decoder receiving VT mid-payload means a malformed stream; best to treat as framing error or emit warning — for Phase 2 keep it simple: VT mid-payload is treated as starting a new frame for robustness, emit MLLP_TRAILING_BYTES for the abandoned partial payload)
  - maxFrameSizeBytes check: if accumulator would exceed limit → throw MllpFramingError('MLLP_FRAME_TOO_LARGE')

- EXPECTING_CR:
  - byte == CR → payload complete; call onFrame with copied payload; transition to SCANNING_FOR_VT
  - byte == LF AND allowLfAfterFs → emit MLLP_LF_AFTER_FS; payload complete; call onFrame; transition to SCANNING_FOR_VT
  - byte == VT (next frame start without CR) AND allowFsOnly → emit MLLP_FS_WITHOUT_CR; payload complete; call onFrame; transition to READING_PAYLOAD
  - Any non-CR byte without tolerance → MllpFramingError('MLLP_FS_WITHOUT_CR') or 'MLLP_LF_AFTER_FS'
  - allowFsOnly means FS alone (without CR) terminates the frame
</pitfall_ref>

    Basic framing:
    - new FrameReader({ onFrame: fn }); reader.push(Buffer.from([0x0B, 0x41, 0x42, 0x1C, 0x0D])) → onFrame called once with Buffer([0x41, 0x42])

    - 1 frame split across N 1-byte chunks → onFrame called once after all chunks pushed
    - Chunk ends mid-delimiter (e.g., chunk ends with [0x1C], next chunk starts with [0x0D]) → correct frame delivered
    - Empty payload [VT, FS, CR]: onFrame called with Buffer.alloc(0); MLLP_EMPTY_PAYLOAD warning emitted
    - Back-to-back frames with zero bytes between them: both delivered correctly

    Byte offset tracking (FRAME-06):
    - After pushing a single 5-byte frame [VT, payload(3), FS, CR], byteOffset reflects 5 bytes processed
    - Warning.byteOffset reflects the absolute stream position, not position within the current chunk
    - After reset(), byteOffset resets to 0

    Tolerance: FRAME-07 — allowFsOnly:
    - Without opt-in: push([VT, 0x41, FS, VT]) → throws MllpFramingError('MLLP_FS_WITHOUT_CR') at EXPECTING_CR
    - With allowFsOnly: true → MLLP_FS_WITHOUT_CR warning emitted; onFrame called with payload; next VT starts new frame

    Tolerance: FRAME-08 — allowLfAfterFs:
    - Without opt-in: push([VT, 0x41, FS, LF]) → throws MllpFramingError (FS+LF instead of FS+CR)
    - With allowLfAfterFs: true → MLLP_LF_AFTER_FS warning emitted; onFrame called; transitions to SCANNING_FOR_VT

    Tolerance: FRAME-09 — allowMissingLeadingVt:
    - Without opt-in: push([0x41]) (no leading VT) → throws MllpFramingError('MLLP_MISSING_LEADING_VT')
    - With allowMissingLeadingVt: true → MLLP_MISSING_LEADING_VT warning; byte treated as first payload byte; accumulation continues until FS+CR

    Tolerance: FRAME-10 — allowLeadingWhitespace:
    - Whitespace bytes (0x20 SP, 0x09 TAB, 0x0A LF, 0x0D CR) before VT:
    - Without opt-in: throws MllpFramingError
    - With allowLeadingWhitespace: true → MLLP_LEADING_WHITESPACE warning with byteOffset of first whitespace byte; whitespace skipped; VT then starts frame

    MLLP_TRAILING_BYTES (WARN-05 — always a warning):
    - Bytes between CR and next VT (that are not whitespace with allowLeadingWhitespace) emit MLLP_TRAILING_BYTES
    - Does not throw even without any tolerance opt-in

    MLLP_EMPTY_PAYLOAD (WARN-05 — always a warning):
    - [VT, FS, CR]: onFrame called with empty Buffer; MLLP_EMPTY_PAYLOAD warning emitted
    - Does not throw even in strict mode

    maxFrameSizeBytes (FRAME-11, WARN-09):
    - Default is 16 MiB; exceeding throws MllpFramingError('MLLP_FRAME_TOO_LARGE')
    - Custom: new FrameReader({ onFrame: fn, maxFrameSizeBytes: 100 }) — accumulating 101 bytes throws
    - Error.byteOffset === the offset at which the cap was reached (not 0)
    - After throw, push() must not be called again on the same reader (state is undefined post-throw)

    - Throwing onWarning handler does not corrupt FSM state; subsequent push() calls work correctly

    - Clears accumulator so partial frame state is discarded
    - Resets byteOffset to 0
    - After reset(), reader can receive a fresh frame correctly

    - onFrame payload is a copied Buffer, not a view into the internal accumulator
    - Pushing more data after a frame fires does not mutate previously delivered payload

    ```typescript
    /**
     * MLLP frame decoder — stateful 3-state FSM for chunked byte-stream parsing.
     *
     * Feed raw TCP chunks via `push(chunk)`. Complete MLLP frames fire synchronously
     * via the `onFrame` callback during each `push()` call.
     *
     * The decoder is **strict by default**: any framing deviation throws `MllpFramingError`
     * unless the matching tolerance option is explicitly enabled (Postel's Law — liberal receiver
     * means explicit tolerance, not silent acceptance).
     *
     * @example
     * ```typescript
     * import { FrameReader } from '@cosyte/hl7-mllp';
     * const reader = new FrameReader({
     *   onFrame: (payload) => handleMessage(payload),
     *   onWarning: (w) => logger.warn(w),
     *   allowFsOnly: true,
     * });
     * socket.on('data', (chunk) => reader.push(chunk));
     * socket.on('close', () => reader.reset());
     * ```
     *
     * @packageDocumentation
     */

    import { VT, FS, CR, LF, DEFAULT_MAX_FRAME_SIZE } from './constants.js';
    import { MllpFramingError } from './error.js';
    import { createWarning } from './registry.js';
    import type { MllpWarning } from './registry.js';

    /** Internal FSM states. */
    type ReaderState = 'SCANNING_FOR_VT' | 'READING_PAYLOAD' | 'EXPECTING_CR';

    /**
     * Options for `FrameReader`.
     *
     * All tolerance opts default to `false` — every framing deviation throws unless
     * explicitly enabled. Server-level defaults (allowFsOnly, allowLfAfterFs, allowLeadingWhitespace)
     * are applied by Phase 4 when constructing readers per SERVER-12.
     *
     * @example
     * ```typescript
     * const opts: FrameReaderOptions = {
     *   onFrame: (p) => process(p),
     *   onWarning: (w) => logger.warn(w),
     *   maxFrameSizeBytes: 4 * 1024 * 1024, // 4 MiB limit
     *   allowFsOnly: true,
     *   allowLfAfterFs: true,
     * };
     * ```
     */
    export interface FrameReaderOptions {
      /** Called synchronously during `push()` for each complete MLLP payload. */
      onFrame: (payload: Buffer) => void;
      /**
       * Called for each tolerated framing deviation. Wrapped in try/catch per WARN-06.
       * A throwing handler will not corrupt FSM state.
       */
      onWarning?: (w: MllpWarning) => void;
      /**
       * Maximum accumulated payload bytes before `MllpFramingError('MLLP_FRAME_TOO_LARGE')`.
       * Default: 16 MiB (FRAME-11 DoS prevention).
       */
      maxFrameSizeBytes?: number;
      /** FRAME-07: Tolerate FS without trailing CR; emits `MLLP_FS_WITHOUT_CR`. */
      allowFsOnly?: boolean;
      /** FRAME-08: Tolerate FS+LF instead of FS+CR; emits `MLLP_LF_AFTER_FS`. */
      allowLfAfterFs?: boolean;
      /** FRAME-09: Tolerate missing leading VT; emits `MLLP_MISSING_LEADING_VT`. */
      allowMissingLeadingVt?: boolean;
      /** FRAME-10: Tolerate SP/TAB/LF/CR before VT; emits `MLLP_LEADING_WHITESPACE`. */
      allowLeadingWhitespace?: boolean;
    }

    /** Initial and growth size for the internal payload accumulator. */
    const INITIAL_ACCUMULATOR_SIZE = 4096;

    /**
     * Stateful MLLP frame decoder. Feed raw TCP byte chunks via `push(chunk)`.
     * Complete frames fire synchronously via `onFrame` callback during `push()`.
     *
     * @example
     * ```typescript
     * const reader = new FrameReader({ onFrame: (payload) => console.log(payload) });
     * socket.on('data', (chunk) => reader.push(chunk));
     * ```
     */
    export class FrameReader {
      private readonly _opts: FrameReaderOptions;
      private readonly _maxFrameSize: number;

      private _state: ReaderState = 'SCANNING_FOR_VT';
      /** Reusable accumulator buffer — grown as needed. */
      private _accumulator: Buffer = Buffer.allocUnsafe(INITIAL_ACCUMULATOR_SIZE);
      /** Number of payload bytes written to accumulator in current frame. */
      private _writePos = 0;
      /** Monotonic absolute stream byte offset. */
      private _byteOffset = 0;
      /** Byte offset where current whitespace run started (for MLLP_LEADING_WHITESPACE). */
      private _wsStart = 0;
      /** Count of leading whitespace bytes in current run. */
      private _wsCount = 0;

      constructor(opts: FrameReaderOptions) {
        this._opts = opts;
        this._maxFrameSize =
          opts.maxFrameSizeBytes !== undefined
            ? opts.maxFrameSizeBytes
            : DEFAULT_MAX_FRAME_SIZE;
      }

      /**
       * Feed a chunk of raw TCP bytes into the FSM. Frames fire synchronously via `onFrame`
       * during this call. May throw `MllpFramingError` for unrecoverable framing violations
       * when the matching tolerance is not enabled.
       */
      push(chunk: Buffer): void {
        for (let i = 0; i < chunk.length; i++, this._byteOffset++) {
          const byte = chunk[i];
          if (byte === undefined) break; // noUncheckedIndexedAccess guard

          this._processByte(byte);
        }
      }

      /**
       * Clear internal accumulator and reset byte offset to 0.
       * Call on reconnect / connection reuse to start fresh (D-03).
       */
      reset(): void {
        this._state = 'SCANNING_FOR_VT';
        this._writePos = 0;
        this._byteOffset = 0;
        this._wsStart = 0;
        this._wsCount = 0;
      }

      private _processByte(byte: number): void {
        switch (this._state) {
          case 'SCANNING_FOR_VT':
            this._scanForVt(byte);
            break;
          case 'READING_PAYLOAD':
            this._readPayload(byte);
            break;
          case 'EXPECTING_CR':
            this._expectCr(byte);
            break;
        }
      }

      private _scanForVt(byte: number): void {
        if (byte === VT) {
          // Normal case: found frame start
          this._wsStart = 0;
          this._wsCount = 0;
          this._state = 'READING_PAYLOAD';
          return;
        }

        // Whitespace before VT
        const isWhitespace = byte === 0x20 || byte === 0x09 || byte === LF || byte === CR;
        if (isWhitespace) {
          if (this._opts.allowLeadingWhitespace === true) {
            if (this._wsCount === 0) {
              this._wsStart = this._byteOffset;
            }
            this._wsCount++;
            return;
          }
          // Whitespace without tolerance is a framing error (treated as MLLP_MISSING_LEADING_VT)
          const snippet = Buffer.from([byte]);
          throw new MllpFramingError(
            'MLLP_MISSING_LEADING_VT',
            this._byteOffset,
            snippet,
            `Expected VT (0x0B) to start MLLP frame, got 0x${byte.toString(16).padStart(2, '0')} at offset ${this._byteOffset}`,
          );
        }

        // Non-VT, non-whitespace byte in SCANNING_FOR_VT
        // If allowLeadingWhitespace was active and we got here, emit that warning first
        if (this._wsCount > 0) {
          this._emitWarning(
            'MLLP_LEADING_WHITESPACE',
            this._wsStart,
            `${this._wsCount} leading whitespace byte(s) before VT at offset ${this._wsStart}`,
          );
          this._wsStart = 0;
          this._wsCount = 0;
        }

        if (this._opts.allowMissingLeadingVt === true) {
          // Treat this byte as the start of a payload
          this._emitWarning(
            'MLLP_MISSING_LEADING_VT',
            this._byteOffset,
            `Missing leading VT (0x0B) — treating byte 0x${byte.toString(16).padStart(2, '0')} at offset ${this._byteOffset} as payload start`,
          );
          this._state = 'READING_PAYLOAD';
          this._appendByte(byte);
          return;
        }

        // Default: framing error
        const snippet = Buffer.from([byte]);
        throw new MllpFramingError(
          'MLLP_MISSING_LEADING_VT',
          this._byteOffset,
          snippet,
          `Expected VT (0x0B) to start MLLP frame, got 0x${byte.toString(16).padStart(2, '0')} at offset ${this._byteOffset}`,
        );
      }

      private _readPayload(byte: number): void {
        if (byte === FS) {
          // End-of-payload marker
          if (this._writePos === 0) {
            // Empty payload — always a warning (WARN-05), never an error
            this._emitWarning(
              'MLLP_EMPTY_PAYLOAD',
              this._byteOffset,
              `Empty payload between VT and FS at offset ${this._byteOffset}`,
            );
          }
          this._state = 'EXPECTING_CR';
          return;
        }

        if (byte === VT) {
          // VT mid-payload: treat as abandoned frame + new frame start
          // Emit MLLP_TRAILING_BYTES for the partial/abandoned payload (always a warning)
          this._emitWarning(
            'MLLP_TRAILING_BYTES',
            this._byteOffset,
            `Unexpected VT (0x0B) mid-payload at offset ${this._byteOffset}; discarding ${this._writePos} accumulated bytes`,
          );
          this._writePos = 0;
          this._state = 'READING_PAYLOAD';
          return;
        }

        // Normal payload byte — check size cap
        if (this._writePos >= this._maxFrameSize) {
          const snippet = Buffer.from(
            this._accumulator.subarray(
              Math.max(0, this._writePos - 32),
              Math.min(this._writePos, this._writePos),
            ),
          );
          throw new MllpFramingError(
            'MLLP_FRAME_TOO_LARGE',
            this._byteOffset,
            snippet,
            `Frame payload exceeded maxFrameSizeBytes (${this._maxFrameSize}) at offset ${this._byteOffset}`,
          );
        }

        this._appendByte(byte);
      }

      private _expectCr(byte: number): void {
        if (byte === CR) {
          // Normal frame completion
          this._deliverFrame();
          this._state = 'SCANNING_FOR_VT';
          return;
        }

        if (byte === LF) {
          if (this._opts.allowLfAfterFs === true) {
            this._emitWarning(
              'MLLP_LF_AFTER_FS',
              this._byteOffset,
              `LF (0x0A) after FS instead of CR (0x0D) at offset ${this._byteOffset}`,
            );
            this._deliverFrame();
            this._state = 'SCANNING_FOR_VT';
            return;
          }
          const snippet = Buffer.from([byte]);
          throw new MllpFramingError(
            'MLLP_LF_AFTER_FS',
            this._byteOffset,
            snippet,
            `Expected CR (0x0D) after FS, got LF (0x0A) at offset ${this._byteOffset}. Enable allowLfAfterFs to tolerate.`,
          );
        }

        if (byte === VT) {
          // Next frame start without CR after FS — allowFsOnly tolerance
          if (this._opts.allowFsOnly === true) {
            this._emitWarning(
              'MLLP_FS_WITHOUT_CR',
              this._byteOffset,
              `FS not followed by CR at offset ${this._byteOffset}; next frame VT found immediately`,
            );
            this._deliverFrame();
            this._state = 'READING_PAYLOAD'; // already consumed VT
            return;
          }
          const snippet = Buffer.from([byte]);
          throw new MllpFramingError(
            'MLLP_FS_WITHOUT_CR',
            this._byteOffset,
            snippet,
            `Expected CR (0x0D) after FS, got VT (0x0B) at offset ${this._byteOffset}. Enable allowFsOnly to tolerate.`,
          );
        }

        // Any other byte after FS without tolerance
        if (this._opts.allowFsOnly === true) {
          // Non-CR, non-LF, non-VT after FS — treat as trailing bytes, frame complete
          this._emitWarning(
            'MLLP_FS_WITHOUT_CR',
            this._byteOffset,
            `FS not followed by CR at offset ${this._byteOffset}; got 0x${byte.toString(16).padStart(2, '0')}`,
          );
          this._deliverFrame();
          // The stray byte: treat as trailing in next SCANNING_FOR_VT context
          this._state = 'SCANNING_FOR_VT';
          // Emit trailing bytes warning and discard
          this._emitWarning(
            'MLLP_TRAILING_BYTES',
            this._byteOffset,
            `Unexpected byte 0x${byte.toString(16).padStart(2, '0')} after frame terminator at offset ${this._byteOffset}`,
          );
          return;
        }

        const snippet = Buffer.from([byte]);
        throw new MllpFramingError(
          'MLLP_FS_WITHOUT_CR',
          this._byteOffset,
          snippet,
          `Expected CR (0x0D) after FS, got 0x${byte.toString(16).padStart(2, '0')} at offset ${this._byteOffset}`,
        );
      }

      private _appendByte(byte: number): void {
        // Grow accumulator if needed
        if (this._writePos >= this._accumulator.length) {
          const newSize = this._accumulator.length * 2;
          const grown = Buffer.allocUnsafe(newSize);
          this._accumulator.subarray(0, this._writePos).copy(grown);
          this._accumulator = grown;
        }
        this._accumulator[this._writePos++] = byte;
      }

      private _deliverFrame(): void {
        // Copy payload out of accumulator — isolate from future accumulator reuse
        const payload = Buffer.from(this._accumulator.subarray(0, this._writePos));
        this._writePos = 0;
        this._opts.onFrame(payload);
      }

      private _emitWarning(
        code: Parameters<typeof createWarning>[0],
        byteOffset: number,
        message: string,
      ): void {
        if (this._opts.onWarning !== undefined) {
          const warning = createWarning(code, byteOffset, message);
          try {
            this._opts.onWarning(warning);
          } catch {
            // WARN-06: throwing handler must not corrupt FSM state
          }
        }
      }
    }
    ```
