const closePromises = [...this._connections].map((conn) => conn.close({ drainTimeoutMs }));

Key implementation note for `_drainAll`: the stragglers loop inside the timeout must iterate `this._connections` (the live Set), not the snapshotted array — because connections may have already closed during the drain window and been removed from the Set. This avoids calling `destroy()` on an already-CLOSED connection.

Actually calling `conn.destroy()` on a CLOSED connection is safe (Connection.destroy() is idempotent via the LEGAL_TRANSITIONS guard), but it is cleaner to check `this._connections` (the live Set) which already only contains non-closed connections.

For the `Promise.all` coordination: since `conn.close()` returns a Promise that resolves when the connection reaches DISCONNECTED or CLOSED, and the 'close' event removes from `_connections`, `Promise.all(closePromises)` resolving means all connections have drained. The timeout is the safety net.

| close() drainTimeoutMs | Caller-supplied integer; uncapped could starve shutdown |
| AbortSignal source | External signal from caller; abort at any point including mid-drain |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Denial of Service | drainTimeoutMs = 0 or undefined | accept | Default is 30_000 ms; caller setting 0 gets immediate destroy of all connections, which is safe behaviour (fast-close) |
| T-04-03-02 | Denial of Service | signal handler accumulation across test instances | mitigate | AbortSignal handler uses `{ once: true }` and is removed via `removeEventListener` on resolution — no accumulation even if listen/close called many times in test suites |
| T-04-03-03 | Denial of Service | slow-close: connections that never drain | mitigate | drainTimeoutMs deadline with `handle.unref()` ensures process can still exit; `conn.destroy()` force-closes stragglers |
| T-04-03-04 | Denial of Service | idle timer accumulation | mitigate | Each idle timer is stored in `let idleTimer`; reset on 'message', cleared on 'close'; `.unref()` prevents process keep-alive |
</threat_model>
