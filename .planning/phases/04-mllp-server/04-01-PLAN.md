<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TCP accept → NetTransport | Untrusted inbound connections from any IP; no authentication at this layer |
| NetTransport → FrameReader | Arbitrary byte streams; frame size capped by maxFrameSizeBytes |
| FrameReader → onMessage handler | Decoded payload passed to caller-supplied callback |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Denial of Service | FrameReader per-connection | mitigate | `maxFrameSizeBytes` (default 16 MB) inherited from SERVER_DEFAULT_FRAMING; throws `MLLP_FRAME_TOO_LARGE` before accumulator grows unbounded |
| T-04-01-02 | Denial of Service | `_connections` Set growth | mitigate | Plan 03 adds `drainTimeoutMs` force-close; at Plan 01 scope, connections are removed on 'close' event to prevent leak |
| T-04-01-03 | Spoofing | remoteAddress in MessageMeta | accept | MLLP operates on internal clinical networks; IP-level spoofing is a network-layer concern outside library scope |
| T-04-01-04 | Information Disclosure | MllpWarning.message | mitigate | warning messages contain only byte offsets and codes (WARN-07), never payload bytes or PII |
| T-04-01-05 | Denial of Service | idle timer accumulation in tests | mitigate | all idle timeout handles call `.unref()` so they do not keep the process alive; cleared on connection 'close' |
</threat_model>
