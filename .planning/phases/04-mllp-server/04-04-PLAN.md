Signal handler cleanup pattern:
```typescript
export async function createStarterServer(opts: StarterServerOptions): Promise<MllpServer> {
  const server = createServer({ ... });

  if (opts.handleSignals === true) {
    const sigHandler = () => {
      void server.close().then(() => process.exit(0)).catch(() => process.exit(1));
    };
    process.once('SIGTERM', sigHandler);
    process.once('SIGINT', sigHandler);

    // Clean up handlers on server close so tests don't accumulate listeners.
    // process.once handlers self-remove on first fire; this removes them early if close()
    // is called before any signal fires (the common test path).
    server.once('close', () => {
      process.removeListener('SIGTERM', sigHandler);
      process.removeListener('SIGINT', sigHandler);
    });
  }

  await server.listen(opts.port, opts.host ?? '0.0.0.0');
  return server;
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| createStarterServer → process signals | Signal handler calls process.exit — must use once() to prevent accumulation across test instances; must also be cleaned up on server.close() |
| getStats() → conn.getStats() | Iterates live connections; concurrent close() during iteration is safe since Set.forEach/for-of is snapshot-based in JS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Denial of Service | SIGTERM handler accumulation in tests | mitigate | `process.once` (not `process.on`) self-removes on first fire; additionally `server.once('close', cleanup)` removes handlers early when close() is called before signal fires — `process.listenerCount('SIGTERM') === 0` after server.close() |
| T-04-04-02 | Elevation of Privilege | process.exit(0) in sigHandler | accept | SIGTERM handler is the documented and expected behaviour for k8s/systemd-managed processes; operator explicitly opts in via `handleSignals: true` |
| T-04-04-03 | Information Disclosure | server.getStats() totalBytesIn/Out | accept | byte counters are non-sensitive operational metrics; no payload content is included |
| T-04-04-04 | Denial of Service | getStats() during high-connection-churn | accept | for-of over a Set is safe during concurrent modification in single-threaded Node; worst case: counts are slightly stale, which is acceptable for operational metrics |
| T-04-04-05 | Tampering | Subscriber mutating frozen event payloads | mitigate | Object.freeze() on all event payload objects; TypeScript `readonly` fields on all interfaces; runtime mutation attempt silently fails in strict mode or throws in strict JS environments |
</threat_model>
